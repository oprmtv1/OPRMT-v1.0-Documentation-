This changelog documents all notable updates to the OPRMT™ Specification, tooling, governance, and documentation.

The format follows Semantic Versioning (MAJOR.MINOR.PATCH):

MAJOR — Breaking changes to the structure, terminology, or rules

MINOR — Additions or improvements that do not break existing specifications

PATCH — Fixes, clarifications, or documentation updates


All changes must be submitted via PR and approved under OPRMT™ governance protocols.

Release Process

1. Preparing a Release

Maintainers must:

1. Review all pending changes


2. Assign each change to MAJOR, MINOR, or PATCH


3. Update this CHANGELOG under a new version heading


4. Update version numbers in:

Specification files

Package definitions (if applicable)

Documentation references




2. Tagging

Once updates are complete:

Tag the commit using vX.X.X

Create a release in GitHub

Attach release notes generated from the changelog


3. Documentation Sync

After tagging:

Update the website or external docs (if applicable)

Notify the OPRMT™ Community

Close all related issues

Remove entries from the “Unreleased” section



---

Versioning Rules

MAJOR

Triggered if changes:

Alter the OPRMT™ structure (Objective → Parameters → Results → Method → Tools)

Modify core definitions

Remove or rename required fields

Break backward compatibility with existing OPRMT™ templates


MINOR

Triggered if changes:

Add new optional fields

Expand examples

Introduce new official templates

Improve existing definitions without breaking them


PATCH

Triggered if changes:

Fix typos

Improve readability

Correct formatting

Update non-structural parts of documentation

Patch security or compliance issues
